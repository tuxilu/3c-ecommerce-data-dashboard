## 🎯 项目概述

这是一个基于 Vue 3 + Vite + ECharts 的现代化电商数据可视化大屏项目，专为3C数码电商场景设计。项目采用暗色科技风格，提供实时数据监控、多维度数据分析和直观的可视化展示。

### 核心特点

- **实时数据监控**：每5秒自动刷新数据，展示最新业务指标
- **多维度分析**：涵盖销售、订单、流量、区域、品类等多个维度
- **响应式设计**：适配不同屏幕尺寸，支持全屏模式
- **炫酷视觉效果**：渐变边框动画、玻璃拟态设计、流畅交互体验
- **高性能渲染**：使用 ECharts 和 Vue 3 组合式 API，确保流畅体验

## 🛠 技术栈

### 前端框架
- **Vue 3.5.30**：使用 Composition API 和 `<script setup>` 语法
- **Vite 8.0.0**：极速构建工具，提供快速的开发体验
- **Pinia 3.0.4**：Vue 3 官方推荐的状态管理库

### 可视化组件
- **ECharts 6.0.0**：强大的图表库，支持多种图表类型

### 样式处理
- **Sass 1.98.0**：CSS 预处理器，提供变量、嵌套等功能

### 网络请求
- **Axios 1.13.6**：HTTP 客户端，用于数据获取

### 构建工具
- **@vitejs/plugin-vue 6.0.5**：Vue 3 的 Vite 插件

## ✨ 功能特性

### 1. 核心指标监控
- **今日销售额**：实时显示当日销售金额，包含同比数据
- **今日订单量**：展示当日订单总数，提供趋势参考

### 2. 销售趋势分析
- **近7天销售趋势**：折线图展示销售数据变化趋势
- **用户访问趋势**：PV/UV 双指标折线图，分析用户行为

### 3. 订单来源分析
- **订单来源占比**：环形图展示不同渠道的订单分布
- 支持移动端、PC端、小程序等多渠道数据

### 4. 商品分类分析
- **商品分类销量**：柱状图展示各分类商品的销售表现
- 支持按销量排序和对比分析

### 5. 区域订单分布
- **各地区订单分布**：中国地图可视化展示区域订单数据
- 支持省份级别的数据展示和交互

### 6. 时间筛选功能
- **今日**：查看当日实时数据
- **近7天**：查看最近一周的数据趋势
- **近30天**：查看月度数据表现

### 7. 全屏模式
- 支持一键切换全屏，提供沉浸式数据查看体验

## 🚀 快速开始

### 环境要求

- Node.js >= 16.0.0
- npm >= 8.0.0 或 yarn >= 1.22.0

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

项目将在 `http://localhost:5173` 启动（如果端口被占用，会自动选择其他端口）。

### 构建生产版本

```bash
npm run build
```

构建后的文件将输出到 `dist` 目录。

### 预览生产版本

```bash
npm run preview
```

## 📖 使用说明

### 基本操作

#### 1. 查看数据大屏

启动项目后，在浏览器中访问 `http://localhost:5173` 即可看到完整的数据大屏。

#### 2. 切换时间范围

点击顶部的时间筛选按钮：
- **今日**：查看当日实时数据
- **近7天**：查看最近一周的数据趋势
- **近30天**：查看月度数据表现

#### 3. 全屏模式

点击右上角的"全屏模式"按钮，切换到全屏显示。再次点击可退出全屏。

#### 4. 图表交互

- **折线图**：鼠标悬停可查看具体数据点，点击图例可显示/隐藏数据系列
- **柱状图**：鼠标悬停显示详细数据，支持点击交互
- **环形图**：鼠标悬停显示扇区详情，点击图例可筛选数据
- **地图**：支持缩放、拖拽，鼠标悬停显示省份详细信息

### 数据刷新

系统默认每5秒自动刷新一次数据，确保显示最新的业务指标。刷新间隔可以在 `src/views/Dashboard.vue` 中修改：

```javascript
// 修改刷新间隔（单位：毫秒）
const timer = setInterval(() => {
  dataStore.fetchAllData()
}, 5000) // 当前设置为5秒
```

## 🔧 自定义和扩展

### 添加新的图表类型

1. 在 `src/components/EchartsChart.vue` 中添加新的图表配置函数：

```javascript
// 雷达图配置
const buildRadarOption = (data, config) => ({
  radar: {
    indicator: data.indicator || [],
    // ... 其他配置
  },
  series: [{
    type: 'radar',
    data: data.values || []
  }],
  ...config
})
```

2. 在 `updateChart` 函数中添加对应的处理逻辑：

```javascript
if (props.chartType === 'radar') {
  option = buildRadarOption(props.chartData, props.chartConfig)
}
```

3. 在页面中使用：

```vue
<EchartsChart
  chart-type="radar"
  :chart-data="yourRadarData"
/>
```

### 添加新的数据指标

1. 在 `src/stores/dataStore.js` 中添加新的状态：

```javascript
const newMetric = ref(null)

export const useDataStore = defineStore('data', () => {
  // ... 其他代码
  
  return {
    newMetric,
    // ... 其他返回值
  }
})
```

2. 在 `src/utils/api.js` 中添加对应的接口：

```javascript
export const fetchNewMetric = () => {
  return request.get('/new-metric')
```

3. 在页面中使用新的数据：


## ❓ 常见问题

### 1. 图表不显示或显示异常

**问题原因**：图表容器尺寸不正确或数据格式错误

**解决方案**：
- 确保图表容器的父元素有明确的高度和宽度
- 检查传入图表组件的数据格式是否正确
- 查看浏览器控制台是否有错误信息

### 2. 地图不显示

**问题原因**：地图数据未正确加载或省份名称不匹配

**解决方案**：
- 确保 `src/assets/china.json` 文件存在
- 检查省份名称映射是否正确
- 查看控制台是否有地图注册错误

### 3. 样式不生效

**问题原因**：样式优先级问题或构建缓存

**解决方案**：
- 清除浏览器缓存
- 重启开发服务器
- 检查 CSS 选择器优先级
- 确保样式文件正确导入

### 4. 数据不更新

**问题原因**：接口请求失败或数据格式不匹配

**解决方案**：
- 检查网络请求是否成功
- 查看接口返回的数据格式是否正确
- 确认时间筛选参数是否正确传递

### 5. 响应式布局问题

**问题原因**：CSS Grid 或 Flexbox 配置不当

**解决方案**：
- 检查媒体查询断点是否正确
- 确保容器有最小高度设置
- 调整 Grid 布局的行列配置

### 6. 性能问题

**问题原因**：频繁的数据更新或图表重绘

**解决方案**：
- 使用防抖和节流优化事件处理
- 减少不必要的数据更新频率
- 优化图表配置，减少渲染复杂度
